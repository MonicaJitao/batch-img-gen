# 参考图与 `/v1/videos` 链路验收清单

在运行 `uvicorn server:app --reload --host 127.0.0.1 --port 8765` 后，用浏览器开发者工具 **Network** 面板核对请求。

## 1. 无参考图（纯文本）

1. 不上传任何参考图。
2. 数量选 1 或 3，填写提示词，点击生成。
3. 确认存在 `POST /v1/videos`，`Content-Type` 为 `multipart/form-data`。
4. Form Data 中含 `model`、`prompt`、`size`（冒号比例，如 `3:4`），**不含** `input_reference`。
5. 随后有 `GET /v1/videos/<id>` 轮询，任务 `status` 最终为 `completed`，结果 URL 来自 `video_url`（或兼容字段）。

## 2. 单张参考图

1. 上传 1 张风格或构图明显的参考图。
2. 提示词明确要求「参考上传图片的色调/构图/主体」。
3. 确认每个槽位对应的 `POST /v1/videos` 中均有 **一条** `input_reference`（文件类型）。
4. 生成结果应能体现参考图约束（具体程度因模型与提示词而异）。

## 3. 多张参考图

1. 上传 2～4 张不同参考图。
2. 提示词要求融合（例如图 1 人物 + 图 2 场景）。
3. 确认每个 `POST /v1/videos` 请求里出现 **多条** 同名字段 `input_reference`（multipart 允许多文件同字段名）。

## 4. 比例映射

1. 在比例下拉里分别选择 `1x1`、`3x4`、`16x9` 等。
2. 每次生成时检查 `POST /v1/videos` 的 `size` 字段为 **`1:1`、`3:4`、`16:9`**（冒号），而不是 `1x1`。

## 5. 模型默认值

1. 清空 Model 输入或首次进入页面时，应默认使用 `gemini-3-pro-image-preview-async`（或你在配置里保存的合法 `*-async` 名）。
2. 若本地存储里仍是旧的同步名 `gemini-3-pro-image-preview`，页面加载后应自动规范化为对应的 `*-async` 名称。

## 6. 大批量 + 参考图（风险提示）

1. 数量选 10～20，且上传多张参考图。
2. 预期：每个并发任务都会重复上传整组参考图，总上传体积大、耗时长；若上游限流或网关超时，应能在槽位错误信息中看到 HTTP/上游返回内容。
